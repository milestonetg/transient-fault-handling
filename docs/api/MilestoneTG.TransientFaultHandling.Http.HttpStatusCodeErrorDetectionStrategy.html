    Remarks

    Forked from Microsoft.Rest.ClientRuntime. In Microsoft's original implementation, 500-Server Error was considered transient. Unless you're running at Netflix scale and velocity, we have found that it is more likely that a HTTP 500 error will not be retry-able within the scope of a retry policy. We recommend HTTP 500 errors be handled with a circuit breaker.
